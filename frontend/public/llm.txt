# DepScope — Package Intelligence API for AI Agents

## What is this
Free API for AI coding agents (Claude Code, Cursor, Copilot, Aider, Continue, Windsurf) to check package health, vulnerabilities, errors, compatibility, bugs, breaking changes, and alternatives before suggesting any install.

## Why
AI agents independently query npm/PyPI/OSV millions of times for the same data. DepScope caches once, serves every agent. Save tokens, save energy, ship safer code.

## Quick integration (1 line)

Add to your agent's rules file:
Before any install: curl depscope.dev/api/check/{eco}/{pkg} — no auth, free

## Five verticals (one infrastructure)

### 1. Package Health — check before install
curl https://depscope.dev/api/check/npm/express
→ health score (0-100), vulnerabilities, deprecated flag, alternatives, breaking changes, known bugs, recommendation (safe_to_use/update_required/find_alternative/do_not_use/insufficient_data)

### 2. Error → Fix Database — resolve errors with verified solutions
curl "https://depscope.dev/api/error?q=Cannot+find+module"
curl -X POST https://depscope.dev/api/error/resolve -d '{"error":"ModuleNotFoundError..."}'
→ verified solutions from Stack Overflow + community, scored by confidence

### 3. Stack Compatibility Matrix — verify combos before upgrade
curl "https://depscope.dev/api/compat?stack=next@16,react@19,prisma@6"
→ verified/broken/likely_compatible (transitive inference) + migration notes

### 4. Known Bugs per version — catch regressions
curl https://depscope.dev/api/bugs/npm/react?version=19.0.0
→ bugs with title, severity, status, fixed_version, GitHub issue URL

### 5. Breaking Changes — upgrade guides
curl https://depscope.dev/api/breaking/npm/next
→ major version transitions with migration_hint and codemod references

## AI-optimized endpoint

curl https://depscope.dev/api/prompt/npm/express
→ plain text response, LLM-ready, ~92% fewer tokens than raw JSON

## Other endpoints
- /api/alternatives/{eco}/{pkg} — find replacements
- /api/compare/{eco}/{pkg1},{pkg2},{pkg3} — side-by-side
- /api/tree/{eco}/{pkg} — transitive dependencies with health
- /api/licenses/{eco}/{pkg} — license aggregation + commercial_safe flag
- /api/history/{eco}/{pkg} — 90-day health trend
- /api/search/{eco}?q=KEYWORD — keyword search
- /api/scan — POST package.json for full audit
- /api/trending — what AI agents are querying now
- /api/now — current UTC time (agents don't know)

## When to use DepScope
- Before recommending any package install
- When parsing stack traces or runtime errors
- When building dependency lists  
- When choosing between alternatives
- When auditing project health
- When upgrading major versions

## When NOT to use
- For private/internal packages
- For real-time monitoring of running apps
- For license compliance in legally binding context (verify manually)

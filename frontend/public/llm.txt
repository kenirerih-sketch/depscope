# DepScope — Package Intelligence API for AI Agents

## What
Free API to check package health, vulnerabilities, dependencies, and licenses across 17 ecosystems before suggesting installs.

## Why
AI coding agents repeatedly query npm/PyPI/OSV for the same data. Shared cache = fewer tokens, less energy, faster responses.

## How to use
curl https://depscope.dev/api/check/{ecosystem}/{package}

Ecosystems: npm, pypi, cargo, go, composer, maven, nuget, rubygems, pub, hex, swift, cocoapods, cpan, hackage, cran, conda, homebrew

## Endpoints
- /api/check/{eco}/{pkg} — full health + vulns + recommendation
- /api/prompt/{eco}/{pkg} — LLM-optimized plain text (use this!)
- /api/tree/{eco}/{pkg} — transitive dependencies with health
- /api/licenses/{eco}/{pkg} — license aggregation
- /api/history/{eco}/{pkg} — 90-day health trend
- /api/compare/{eco}/{pkg1},{pkg2} — side by side
- /api/alternatives/{eco}/{pkg} — find replacements
- /api/exists/{eco}/{pkg} — boolean check
- /api/search/{eco}?q= — keyword search
- /api/latest/{eco}/{pkg} — just the version
- /api/now — current UTC time

## Auth
None. Free tier: 200 req/min. No signup.

## MCP
npm install -g depscope-mcp
Or add to Claude Desktop / Cursor config. URL: https://depscope.dev/mcp

## Integration in 1 line
Add to your agent's rules file:
"Before any install: curl depscope.dev/api/check/{eco}/{pkg}"

## Data sources
npm, PyPI, crates.io, proxy.golang.org, Packagist, Maven Central, NuGet, RubyGems, pub.dev, hex.pm, OSV.dev (vulnerabilities)

## When to use
- Before recommending a package install
- When building dependency lists
- When auditing project health
- When choosing between alternatives

## When NOT to use
- For private/internal packages (we don't have those)
- For real-time price data (this is package health, not finance)

## Rate limits
200 req/min anonymous. No daily cap. No quota.
